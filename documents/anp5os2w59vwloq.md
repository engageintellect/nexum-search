content: "<p><strong>Next.js</strong> is an <a class=\"mw-redirect\" title=\"Open-source\" href=\"https://en.wikipedia.org/wiki/Open-source\">open-source</a> web development framework created by the private company <a title=\"Vercel\" href=\"https://en.wikipedia.org/wiki/Vercel\">Vercel</a> providing <a class=\"mw-redirect\" title=\"React (JavaScript library)\" href=\"https://en.wikipedia.org/wiki/React_(JavaScript_library)\">React</a>-based web applications with <a title=\"Server-side scripting\" href=\"https://en.wikipedia.org/wiki/Server-side_scripting\">server-side rendering</a> and <a title=\"Static web page\" href=\"https://en.wikipedia.org/wiki/Static_web_page\">static website</a> generation.</p>\r\n<p>React documentation mentions Next.js among \"Recommended Toolchains\" advising it to developers when \"building a server-rendered website with Node.js\".<sup id=\"cite_ref-React_documentation_5-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-React_documentation-5\">[5]</a></sup> Where traditional React apps can only render their content in the client-side browser, Next.js extends this functionality to include applications rendered on the server-side.</p>\r\n<p>The copyright and trademarks for Next.js are owned by Vercel,<sup id=\"cite_ref-6\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-6\">[6]</a></sup> which also maintains and leads its open-source development.<sup id=\"cite_ref-7\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-7\">[7]</a></sup></p>\r\n<h2><span id=\"Background\" class=\"mw-headline\">Background</span></h2>\r\n<p>Next.js is a <a class=\"mw-redirect\" title=\"React (web framework)\" href=\"https://en.wikipedia.org/wiki/React_(web_framework)\">React</a> framework that enables several extra features, including <a title=\"Server-side scripting\" href=\"https://en.wikipedia.org/wiki/Server-side_scripting\">server-side rendering</a> and generating <a title=\"Static web page\" href=\"https://en.wikipedia.org/wiki/Static_web_page\">static websites</a>.<sup id=\"cite_ref-compareStatic_8-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-compareStatic-8\">[8]</a></sup> React is a <a title=\"JavaScript library\" href=\"https://en.wikipedia.org/wiki/JavaScript_library\">JavaScript library</a> that is traditionally used to build web applications rendered in the client's browser with JavaScript.<sup id=\"cite_ref-thakkar1_9-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar1-9\">[9]</a></sup> Developers recognize several problems with this strategy however, such as not catering to users who do not have access to JavaScript or have disabled it, potential security issues, significantly extended page loading times, and harm to the site's overall <a title=\"Search engine optimization\" href=\"https://en.wikipedia.org/wiki/Search_engine_optimization\">search engine optimization</a>.<sup id=\"cite_ref-thakkar1_9-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar1-9\">[9]</a></sup> Frameworks such as Next.js sidestep these problems by allowing some or all of the website to be rendered on the server-side before being sent to the client.<sup id=\"cite_ref-thakkar1_9-2\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar1-9\">[9]</a></sup><sup id=\"cite_ref-thakkar2_10-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar2-10\">[10]</a></sup> Next.js is one of the most popular frameworks for React.<sup id=\"cite_ref-11\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-11\">[11]</a></sup> It is one of several recommended \"toolchains\" available when starting a new app, all of which provide a layer of abstraction to aid in common tasks.<sup id=\"cite_ref-SmashingStyle_12-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-SmashingStyle-12\">[12]</a></sup> Next.js requires <a title=\"Node.js\" href=\"https://en.wikipedia.org/wiki/Node.js\">Node.js</a> and can be initialized using <a class=\"mw-redirect\" title=\"Node Package Manager\" href=\"https://en.wikipedia.org/wiki/Node_Package_Manager\">Node Package Manager</a>.</p>\r\n<p>Google has contributed to the Next.js project, including 43 pull requests in 2019.<sup id=\"cite_ref-insiders_13-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-insiders-13\">[13]</a></sup> As of March 2022, the framework is used by many large websites, including <a title=\"Walmart\" href=\"https://en.wikipedia.org/wiki/Walmart\">Walmart</a>, <a class=\"mw-redirect\" title=\"Apple Inc\" href=\"https://en.wikipedia.org/wiki/Apple_Inc\">Apple</a>, <a title=\"Nike, Inc.\" href=\"https://en.wikipedia.org/wiki/Nike,_Inc.\">Nike</a>, <a title=\"Netflix\" href=\"https://en.wikipedia.org/wiki/Netflix\">Netflix</a>, <a title=\"TikTok\" href=\"https://en.wikipedia.org/wiki/TikTok\">TikTok</a>, <a title=\"Uber\" href=\"https://en.wikipedia.org/wiki/Uber\">Uber</a>, <a title=\"Lyft\" href=\"https://en.wikipedia.org/wiki/Lyft\">Lyft</a>, and <a title=\"Starbucks\" href=\"https://en.wikipedia.org/wiki/Starbucks\">Starbucks</a>.<sup id=\"cite_ref-thakkar1_9-3\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar1-9\">[9]</a></sup> In early 2020, Vercel secured twenty-one million dollars in Series A funding to support improvements to the software.<sup id=\"cite_ref-simplify_1-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-simplify-1\">[1]</a></sup> The framework's original author, Guillermo Rauch, is currently the CEO of Vercel, and the project's lead maintainer is Tim Neutkens.<sup id=\"cite_ref-14\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-14\">[14]</a></sup></p>\r\n<h2><span id=\"Development_history\" class=\"mw-headline\">Development history</span></h2>\r\n<p>Next.js was first released as an open-source project on <a title=\"GitHub\" href=\"https://en.wikipedia.org/wiki/GitHub\">GitHub</a> on October 25, 2016.<sup id=\"cite_ref-originalRelease_3-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-originalRelease-3\">[3]</a></sup> It was originally developed based on six principles: out-of-the-box functionality requiring no setup, JavaScript everywhere, all functions are written in JavaScript, automatic code-splitting and server-rendering, configurable data-fetching, anticipating requests, and simplifying deployment.<sup id=\"cite_ref-15\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-15\">[15]</a></sup></p>\r\n<p>Next.js 2.0 was announced in March 2017 including several improvements that made it easier to work with small websites. It also increased the build efficiency and improved the scalability of the hot-module replacement feature.<sup id=\"cite_ref-16\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-16\">[16]</a></sup></p>\r\n<p>Version 7.0 was released in September 2018 with improved error handling and support for React's context API for improved dynamic route handling. This was also the first version to upgrade to <a title=\"Webpack\" href=\"https://en.wikipedia.org/wiki/Webpack\">Webpack</a> 4.<sup id=\"cite_ref-17\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-17\">[17]</a></sup></p>\r\n<p>Version 8.0 was released in February 2019 and was the first version to offer serverless deployment of applications, in which the code is split up into <a class=\"mw-redirect\" title=\"Lambda function (computer programming)\" href=\"https://en.wikipedia.org/wiki/Lambda_function_(computer_programming)\">lambda functions</a> that are run on demand. The version also reduced the time and resources required for static exports and improved prefetch performance.<sup id=\"cite_ref-18\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-18\">[18]</a></sup></p>\r\n<p>Version 9.3, announced in March 2020, included various optimizations and global <a class=\"mw-redirect\" title=\"Sass (stylesheet language)\" href=\"https://en.wikipedia.org/wiki/Sass_(stylesheet_language)\">Sass</a> and CSS module support.<sup id=\"cite_ref-19\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-19\">[19]</a></sup></p>\r\n<p>On July 27, 2020 Next.js version 9.5 was announced, adding new capabilities including incremental static regeneration, rewrites, and redirect support.<sup id=\"cite_ref-20\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-20\">[20]</a></sup></p>\r\n<p>On June 15, 2021 Next.js version 11 was released, introducing among others: <a title=\"Webpack\" href=\"https://en.wikipedia.org/wiki/Webpack\">Webpack</a> 5 support, preview of real-time collaborative coding functionality \"Next.js Live\", and experimental function of automatic conversion from Create React App to Next.js compatible form \"Create React App Migration\".<sup id=\"cite_ref-21\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-21\">[21]</a></sup></p>\r\n<p>On October 26, 2021, Next.js 12 was released, adding a Rust compiler, making the compilation faster, <a title=\"AVIF\" href=\"https://en.wikipedia.org/wiki/AVIF\">AVIF</a> support, Edge Functions &amp; Middleware, and Native ESM &amp; URL Imports.<sup id=\"cite_ref-22\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-22\">[22]</a></sup></p>\r\n<p>On October 26, 2022, Vercel released Next.js 13. This major release brought about a new routing pattern in <a title=\"Software release life cycle\" href=\"https://en.wikipedia.org/wiki/Software_release_life_cycle\">beta</a>, with the addition of the App Router that includes support for layouts, React Server Components, streaming, and a new set of data fetching methods. Furthermore, Vercel announced a new toolchain for front-end development called Turbo, including Turbopack as a successor to Webpack, Turborepo for incremental build systems.<sup id=\"cite_ref-23\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-23\">[23]</a></sup></p>\r\n<p>In May 2023, Vercel released Next.js 13.4. This brought with it the stable version of App Router, which allows developers to use it in production.<sup id=\"cite_ref-24\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-24\">[24]</a></sup></p>\r\n<h2><span id=\"Styling_and_features\" class=\"mw-headline\">Styling and features</span></h2>\r\n<p>Next.js supports styling with <a title=\"CSS\" href=\"https://en.wikipedia.org/wiki/CSS\">CSS</a> as well as precompiled <a class=\"mw-redirect\" title=\"Sass (stylesheet language)\" href=\"https://en.wikipedia.org/wiki/Sass_(stylesheet_language)\">Scss</a> and <a class=\"mw-redirect\" title=\"Sass (stylesheet language)\" href=\"https://en.wikipedia.org/wiki/Sass_(stylesheet_language)\">Sass</a>, <a title=\"CSS-in-JS\" href=\"https://en.wikipedia.org/wiki/CSS-in-JS\">CSS-in-JS</a>, and <a title=\"JSX (JavaScript)\" href=\"https://en.wikipedia.org/wiki/JSX_(JavaScript)\">styled JSX</a>.<sup id=\"cite_ref-SmashingStyle_12-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-SmashingStyle-12\">[12]</a></sup> In addition, it is built with <a title=\"TypeScript\" href=\"https://en.wikipedia.org/wiki/TypeScript\">TypeScript</a> support and smart bundling.<sup id=\"cite_ref-25\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-25\">[25]</a></sup> The open-source <a class=\"mw-redirect\" title=\"Transpiler\" href=\"https://en.wikipedia.org/wiki/Transpiler\">transpiler</a> <a title=\"Babel (transcompiler)\" href=\"https://en.wikipedia.org/wiki/Babel_(transcompiler)\">Babel</a> is used to transform and compile code into JavaScript usable by a browser. <a title=\"Webpack\" href=\"https://en.wikipedia.org/wiki/Webpack\">Webpack</a>, another open-source tool, is used to bundle the modules afterward. All of these tools are used with <a title=\"Npm (software)\" href=\"https://en.wikipedia.org/wiki/Npm_(software)\">npm</a> in a terminal.<sup id=\"cite_ref-insiders_13-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-insiders-13\">[13]</a></sup></p>\r\n<p>The main feature of Next.js is its use of server-side rendering to reduce the burden on web browsers and provide enhanced security<sup class=\"noprint Inline-Template Template-Fact\" style=\"white-space: nowrap;\">[<em><a title=\"Wikipedia:Citation needed\" href=\"https://en.wikipedia.org/wiki/Wikipedia:Citation_needed\"><span title=\"This claim needs references to reliable sources. (August 2021)\">citation needed</span></a></em>]</sup>. This can be done for any part of the application or the entire system, allowing for content-rich pages to be singled out for server-side rendering.<sup id=\"cite_ref-thakkar1_9-4\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar1-9\">[9]</a></sup> It can also be done only for first-time visitors, to reduce the burden on web browsers that have yet to download any of the site's assets.<sup id=\"cite_ref-thakkar2_10-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar2-10\">[10]</a></sup> The \"hot reloading\" feature detects changes as they are made and re-renders the appropriate pages so the server avoids the need to be restarted. This allows changes made to the application code to be immediately reflected in the web browser, though some browsers will require the page to be refreshed.<sup id=\"cite_ref-thakkar1_9-5\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar1-9\">[9]</a></sup> The software uses page-based routing for developer convenience and includes support for dynamic routing. Other features include hot-module replacement so that modules can be replaced live, automatic code splitting, which only includes code necessary to load the page, and page prefetching to reduce load time.<sup id=\"cite_ref-thakkar1_9-6\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-thakkar1-9\">[9]</a></sup></p>\r\n<p>Next.js also supports Incremental Static Regeneration<sup id=\"cite_ref-26\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Next.js#cite_note-26\">[26]</a></sup> and static site generation - a compiled version of the website is usually built during build time and saved as a .next folder. When a user makes a request, the pre-built version which are static HTML pages are cached and sent to them. This makes the load time very fast, but it's not suitable for every website because for interactive sites that change often and utilize a lot of user input will not be suitable.</p>\r\n<h2><span
