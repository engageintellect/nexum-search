content: "<p><strong>Svelte</strong> is a <a class=\"mw-redirect\" title=\"Free and open-source\" href=\"https://en.wikipedia.org/wiki/Free_and_open-source\">free and open-source</a> <a class=\"mw-redirect\" title=\"Front end and back end\" href=\"https://en.wikipedia.org/wiki/Front_end_and_back_end\">front end</a> component framework<sup id=\"cite_ref-rethinking-reactivity_4-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-rethinking-reactivity-4\">[4]</a></sup> or language<sup id=\"cite_ref-5\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-5\">[5]</a></sup> created by Rich Harris and maintained by the Svelte core team members.<sup id=\"cite_ref-infoworld_6-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-infoworld-6\">[6]</a></sup> Svelte is not a monolithic <a title=\"JavaScript\" href=\"https://en.wikipedia.org/wiki/JavaScript\">JavaScript</a> library imported by applications: instead, Svelte <a title=\"Compiler\" href=\"https://en.wikipedia.org/wiki/Compiler\">compiles</a> HTML templates to specialized code that manipulates the <a title=\"Document Object Model\" href=\"https://en.wikipedia.org/wiki/Document_Object_Model\">DOM</a> directly, which may reduce the size of transferred files and give better client performance.<sup id=\"cite_ref-react-etc_7-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-react-etc-7\">[7]</a></sup> Application code is also processed by the compiler, inserting calls to automatically recompute data<sup id=\"cite_ref-rethinking-reactivity_4-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-rethinking-reactivity-4\">[4]</a></sup> and re-render UI elements when the data they depend on is modified.<sup id=\"cite_ref-infoq_8-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-infoq-8\">[8]</a></sup> This also avoids the overhead associated with runtime intermediate representations, such as <a title=\"Virtual DOM\" href=\"https://en.wikipedia.org/wiki/Virtual_DOM\">virtual DOM</a>,<sup id=\"cite_ref-pure-overhead_9-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-pure-overhead-9\">[9]</a></sup> unlike traditional frameworks (such as <a class=\"mw-redirect\" title=\"React (JavaScript library)\" href=\"https://en.wikipedia.org/wiki/React_(JavaScript_library)\">React</a> and <a title=\"Vue.js\" href=\"https://en.wikipedia.org/wiki/Vue.js\">Vue</a>) which carry out the bulk of their work at runtime, <em>i.e.</em> in the browser.<sup id=\"cite_ref-react-etc_7-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-react-etc-7\">[7]</a></sup> <sup id=\"cite_ref-infoq_8-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-infoq-8\">[8]</a></sup> <sup id=\"cite_ref-infoworld_6-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-infoworld-6\">[6]</a></sup> <sup id=\"cite_ref-github_10-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-github-10\">[10]</a></sup> <sup id=\"cite_ref-rethinking-reactivity_4-2\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-rethinking-reactivity-4\">[4]</a></sup> <sup id=\"cite_ref-pure-overhead_9-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-pure-overhead-9\">[9]</a></sup></p>\r\n<p>The compiler itself is written in <a title=\"TypeScript\" href=\"https://en.wikipedia.org/wiki/TypeScript\">TypeScript</a>. Its <a title=\"Source code\" href=\"https://en.wikipedia.org/wiki/Source_code\">source code</a> is licensed under <a title=\"MIT License\" href=\"https://en.wikipedia.org/wiki/MIT_License\">MIT License</a> and hosted on <a title=\"GitHub\" href=\"https://en.wikipedia.org/wiki/GitHub\">GitHub</a>.<sup id=\"cite_ref-github_10-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-github-10\">[10]</a></sup></p>\r\n<h2><span id=\"History\" class=\"mw-headline\">History</span></h2>\r\n<p>The predecessor of Svelte is Ractive.js, which Rich Harris had developed earlier.<sup id=\"cite_ref-:0_11-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-:0-11\">[11]</a></sup></p>\r\n<p>Version 1 of Svelte was written in <a title=\"JavaScript\" href=\"https://en.wikipedia.org/wiki/JavaScript\">JavaScript</a> and was released on 29 November 2016. It was basically Ractive with a compiler.<sup id=\"cite_ref-:1_12-0\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-:1-12\">[12]</a></sup> The name Svelte was chosen by Rich Harris and his coworkers at <em><a title=\"The Guardian\" href=\"https://en.wikipedia.org/wiki/The_Guardian\">The Guardian</a></em>.<sup id=\"cite_ref-:1_12-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-:1-12\">[12]</a></sup></p>\r\n<p>Version 2 of Svelte was released on 19 April 2018. It set out to correct what the maintainers viewed as mistakes in the earlier version such as replacing double curly braces with single curly braces.<sup id=\"cite_ref-:1_12-2\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-:1-12\">[12]</a></sup></p>\r\n<p>Version 3 of Svelte is written in <a title=\"TypeScript\" href=\"https://en.wikipedia.org/wiki/TypeScript\">TypeScript</a> and was released on 21 April 2019. It rethought reactivity by using the compiler to instrument assignments behind the scenes.<sup id=\"cite_ref-rethinking-reactivity_4-3\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-rethinking-reactivity-4\">[4]</a></sup></p>\r\n<p>The SvelteKit web framework was announced in October 2020 and entered beta in March 2021.<sup id=\"cite_ref-13\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-13\">[13]</a></sup><sup id=\"cite_ref-14\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-14\">[14]</a></sup></p>\r\n<p>Key early contributors became involved with Conduitry joining with the release of Svelte 1, Tan Li Hau joining in 2019, and Ben McCann joining in 2020.<sup id=\"cite_ref-:1_12-3\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-:1-12\">[12]</a></sup> Rich Harris and Simon Holthausen joined <a title=\"Vercel\" href=\"https://en.wikipedia.org/wiki/Vercel\">Vercel</a> to work on Svelte fulltime in 2022.<sup id=\"cite_ref-15\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-15\">[15]</a></sup></p>\r\n<p>SvelteKit 1.0 was released in December 2022 after two years in development.<sup id=\"cite_ref-16\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-16\">[16]</a></sup></p>\r\n<h2><span id=\"Syntax\" class=\"mw-headline\">Syntax</span></h2>\r\n<p>Svelte applications and components are defined in <code id=\"\" class=\"mw-highlight mw-highlight-lang-text mw-content-ltr\" dir=\"ltr\">.svelte</code> files, which are <a title=\"HTML\" href=\"https://en.wikipedia.org/wiki/HTML\">HTML</a> files extended with templating syntax that is based on JavaScript and is similar to <a title=\"JSX (JavaScript)\" href=\"https://en.wikipedia.org/wiki/JSX_(JavaScript)\">JSX</a>.</p>\r\n<p>Svelte repurposes JavaScript's native labeled statement syntax <code id=\"\" class=\"mw-highlight mw-highlight-lang-text mw-content-ltr\" dir=\"ltr\">$:</code> to mark <a title=\"Reactive programming\" href=\"https://en.wikipedia.org/wiki/Reactive_programming\">reactive</a> statements. Top-level variables become the component's state and exported variables become the properties that the component receives. It is a <a title=\"Next.js\" href=\"https://en.wikipedia.org/wiki/Next.js\">Next.js</a>-style framework that dramatically reduces the amount of code that gets sent to the browser. The maintainers had previously created Sapper, which was the predecessor of SvelteKit.<sup id=\"cite_ref-18\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-18\">[18]</a></sup></p>\r\n<p>The Svelte maintainers also maintain a number of integrations for popular software projects under the Svelte organization including integrations for <a title=\"Vite (software)\" href=\"https://en.wikipedia.org/wiki/Vite_(software)\">Vite</a>, Rollup, <a title=\"Webpack\" href=\"https://en.wikipedia.org/wiki/Webpack\">Webpack</a>, TypeScript, <a title=\"Visual Studio Code\" href=\"https://en.wikipedia.org/wiki/Visual_Studio_Code\">VS Code</a>, Chrome Developer Tools, <a title=\"ESLint\" href=\"https://en.wikipedia.org/wiki/ESLint\">ESLint</a>, and prettier.<sup id=\"cite_ref-19\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-19\">[19]</a></sup> A number of external projects such as Storybook have also created integrations with Svelte and SvelteKit.</p>\r\n<h2><span id=\"Influence\" class=\"mw-headline\">Influence</span></h2>\r\n<p><a title=\"Vue.js\" href=\"https://en.wikipedia.org/wiki/Vue.js\">Vue.js</a> modeled its API and single-file components after Ractive.js, the predecessor of Svelte.<sup id=\"cite_ref-:0_11-1\" class=\"reference\"><a href=\"https://en.wikipedia.org/wiki/Svelte#cite_note-:0-11\">[11]</a></sup></p>\r\n<h2><span id=\"Adoption\" class=\"mw-headline\">Adoption</span></h2>\r\n<p>Svelte is widely praised by developers.
